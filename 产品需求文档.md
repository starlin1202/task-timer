# DeepFocus（深度聚焦）产品需求文档

## 一、产品概述

### 1.1 产品定位
DeepFocus 是一款帮助用户通过"聚焦单一领域、持续投入时间"来实现自我提升的时间管理应用。核心理念：在一个周期内只聚焦**一个**核心目标，通过每天持续投入时间，累积达到几百甚至上千小时，从而提高成功概率。

### 1.2 目标用户
- **主要人群**：
  - 工作族：下班后利用有限时间学习、创作、提升自我
  - 学生党：短期内有特定考试目标，如考研、考四六级、考公、考证等
- **使用场景**：
  - 工作族：学习新技能、创作内容、健身锻炼、培养爱好
  - 学生党：备考复习、专项突破、知识积累
- **用户痛点**：精力分散在太多领域，无法在一个方向持续积累

### 1.3 产品形态
- **当前版本**：Web 应用（响应式设计，支持移动端浏览器访问）
- **技术架构**：采用可跨端技术栈，为未来开发原生 App 预留扩展空间
- **数据存储**：V1 版本使用 LocalStorage 本地存储，V2 版本增加云端同步

## 二、核心功能模块

### 2.1 任务管理系统

#### 2.1.1 核心任务（最多1个）
| 字段 | 说明 | 必填 |
|------|------|------|
| 任务名称 | 用户自定义，如"学习 Python"、"写作" | 是 |
| 任务周期 | 周 / 月 / 年 | 是 |
| 目标时长 | 周期内计划投入的总小时数 | 是 |
| 每日目标 | 建议每日投入时长（用于参考） | 否 |
| 已累积时长 | 系统自动计算 | - |
| 进度百分比 | 已累积 / 目标时长 | - |
| 创建时间 | 系统自动记录 | - |
| 状态 | 进行中 / 已归档 / 已放弃 | - |

**业务规则**：
- 每个周期内最多创建 **1 个**核心任务
- 周期结束后自动归档，不可继续计时
- 进行中的任务可随时修改目标时长（增加或减少）
- **进行中的任务用户可以选择放弃，放弃后任务进入已放弃状态，不可继续计时，但保留已累积的时间记录**
- 核心任务区域在首页占据最大面积，视觉权重最高

#### 2.1.2 非核心任务（无数量限制）
| 字段 | 说明 | 必填 |
|------|------|------|
| 任务名称 | 用户自定义 | 是 |
| 创建时间 | 系统自动记录 | - |
| 已累积时长 | 系统自动计算 | - |

**业务规则**：
- 无数量限制
- 无周期概念，长期有效
- 无目标时长，首页不展示进度
- 计时不会影响核心任务进度

### 2.2 计时系统

#### 2.2.1 功能描述
- **极简模式**：点击开始计时后，用户可自由切换到其他页面或应用
- **计时精度**：精确到秒，显示格式为 `HH:MM:SS`
- **后台运行**：页面切换到后台或锁屏时，计时继续（通过 Web Worker 或时间戳计算实现）
- **全局计时条**：无论核心任务还是非核心任务，点击开始后，首页顶部出现横条型计时器，展示当前任务名称和实时计时

#### 2.2.2 计时流程
**核心任务计时流程**：
```
用户点击"开始" → 记录开始时间戳 → 显示计时器
                                    ↓
用户点击"暂停" → 暂停计时，记录已计时时长
                                    ↓
用户点击"继续" → 恢复计时
                                    ↓
用户点击"停止" → 计算总时长 → 累加到对应任务 → 更新进度
```

**非核心任务计时流程**：
```
用户点击"开始" → 记录开始时间戳 → 显示计时器
                                    ↓
用户点击"暂停" → 暂停计时，记录已计时时长
                                    ↓
用户点击"继续" → 恢复计时
                                    ↓
用户点击"停止" → 计算总时长 → 累加到对应任务
```

**说明**：
- 核心任务只有一个，不存在多个核心任务同时计时的情况
- 非核心任务之间、非核心任务与核心任务之间计时互不干扰
- 暂停期间不计入时长，用户可自由处理其他事情后恢复计时

#### 2.2.3 修正任务时间
- **入口**：统计页中每个任务统计区块内的"修正时间"按钮
- **功能**：支持增加或减少某任务的时长记录
- **用途**：解决用户忘记启动或停止计时导致的时间统计错误

##### 修正时间弹窗内容

**弹窗标题**：修正「任务名称」的时间

**表单元素**：
| 元素 | 类型 | 说明 |
|------|------|------|
| 日期选择 | 日期选择器 | 默认选中今天，支持选择历史日期 |
| 操作类型 | 单选按钮组 | 增加时间 / 减少时间 |
| 时长输入 | 数字输入框 | 小时（0-23）+ 分钟（0-59） |
| 当前时长显示 | 文本 | 显示该日期已记录的时长，作为参考 |
| 确认按钮 | 按钮 | 提交修正 |
| 取消按钮 | 按钮 | 关闭弹窗 |

**业务规则**：
- **增加时间**：输入需要增加的时长，累加到选定日期
- **减少时间**：输入需要减少的时长，从选定日期扣除
  - 校验规则：减少后的时长不能为负数
  - 错误提示：如果减少时长超过该日期已有时长，提示"减少时长不能超过当前已记录时长"
- **日期范围**：只能选择今天及之前的日期，不能选择未来日期

**弹窗交互**：
- 点击遮罩层或取消按钮，关闭弹窗
- 点击确认按钮，校验通过后执行修正并关闭弹窗
- 修正成功后，统计页数据实时更新

---

### 2.3 统计系统

#### 2.3.1 统计维度
| 维度 | 展示内容 |
|------|----------|
| 按日 | 当天各任务时间分布（圆环图/列表） |
| 按周 | 本周各任务时间分布（圆环图） |
| 按月 | 本月各任务时间分布（圆环图） |

#### 2.3.2 统计内容
- **任务列表**：所有任务（核心任务 + 非核心任务）平等展示，按累积时长由长到短排序
- **单个任务统计项**：
  - 任务名称
  - 累积时长
  - 修正时间按钮（用于调整该任务的时长记录）
- **总投入时长**：所有任务时间总和
- **圆环图**：展示各任务时间占比（按累积时长从长到短排序，不同颜色区分）
- **图表交互**：
  - 悬停显示任务名称和具体时长
  - 点击图例可隐藏/显示对应任务
  - 圆环中心显示总时长

**预置规则**：
- 首次打开应用时自动创建
- 用户可以像普通非核心任务一样使用、删除或拖拽排序
- 预置任务的累积时长从0开始

1. **任务名称**（显示在区块上方或中间）
2. **开始按钮**（▶ 图标，点击开始计时）
3. **拖拽排序手柄**（⋮ 或 ≡ 图标，长按拖动排序）
4. **删除按钮**（✕ 图标，点击删除任务）

#### 非核心任务交互规则
- **开始计时**：点击开始按钮，启动计时器并在全局计时条显示
- **拖拽排序**：长按拖拽手柄，拖动卡片改变顺序，排序结果自动保存
- **删除任务**：点击删除按钮，弹出确认对话框后删除
- **区块样式**：紧凑设计，突出操作按钮，一行最多 3 个

#### 全局计时条元素
1. 状态指示器（🟢 表示计时中，⏸️ 表示暂停）
2. 当前任务名称
3. 实时计时器（HH:MM:SS）
4. 暂停/继续按钮（点击后暂停或恢复计时）
5. 停止按钮（点击后停止计时并保存时长）

## 四、交互流程

### 4.1 首次使用流程
```
进入应用 → 默认显示任务页 → 展示空状态引导 → 用户创建第一个核心任务 → 进入正常使用
```

### 4.2 日常使用时序
```
1. 用户打开应用
2. 查看核心任务进度
3. 选择要进行的任务
4. 点击"开始计时"
5. （用户进行任务，可切换页面/应用）
6. 需要临时中断时，点击"暂停"
7. 处理完其他事情后，点击"继续"
8. 完成任务后，点击"停止计时"
9. 系统自动累加时长并更新进度
```

### 4.3 修正时间流程
```
1. 用户在统计页点击某个任务统计项上的"修正"按钮
2. 弹出修正时间弹窗
3. 选择日期（默认今天）
4. 选择操作类型：增加时间 / 减少时间
5. 输入时长（小时 + 分钟）
6. 系统校验：减少时间时，检查该日期剩余时长是否足够
7. 确认后更新对应日期和任务的累积时长
```

### 4.4 添加非核心任务流程
```
1. 用户点击"其他任务"区域标题右侧的 [+] 按钮
2. 弹出添加非核心任务弹窗
3. 输入任务名称
4. 点击确认按钮
5. 弹窗关闭，新任务区块显示在区域末尾
```

### 4.5 非核心任务排序流程
```
1. 用户长按/按住非核心任务小区块的拖拽手柄（⋮ 或 ≡）
2. 卡片进入拖拽状态，其他卡片让出位置
3. 用户拖动卡片到目标位置
4. 释放后，卡片固定在新位置
5. 系统自动保存新的排序顺序
```

### 4.6 删除非核心任务流程
```
1. 用户点击非核心任务小区块的删除按钮（✕）
2. 弹出确认对话框："确定要删除此任务吗？已记录的时间仍会保留在统计中。"
3. 用户点击确认
4. 任务从"其他任务"区域移除
```

### 4.7 清除统计流程
```
1. 用户在统计页点击底部的"清除统计"按钮
2. 弹出二次确认弹窗：
   - 标题：清除所有统计数据
   - 内容：确定要清除所有统计数据吗？此操作不可恢复，所有任务的累积时长将被清零。
   - 按钮：[取消] [确定清除]
3. 用户点击"确定清除"
4. 系统执行清除操作：
   - 所有任务的累积时长清零
   - 所有时间记录清空
   - 核心任务状态重置（保留任务本身，但进度归零）
5. 统计页数据实时更新，显示空状态
```

## 五、数据模型

### 5.1 核心任务（CoreTask）
```typescript
interface CoreTask {
  id: string;              // 唯一标识
  name: string;            // 任务名称
  period: 'week' | 'month' | 'year';  // 周期类型
  targetHours: number;     // 目标时长（小时）
  dailyGoal?: number;      // 每日目标（小时，可选）
  accumulatedHours: number; // 已累积时长（小时）
  createdAt: number;       // 创建时间戳
  endAt: number;           // 周期结束时间戳
  status: 'active' | 'archived';  // 状态
}
```

### 5.2 非核心任务（SideTask）
```typescript
interface SideTask {
  id: string;              // 唯一标识
  name: string;            // 任务名称
  accumulatedHours: number; // 已累积时长（小时）
  sortOrder: number;       // 排序序号，用于拖拽排序
  createdAt: number;       // 创建时间戳
}
```

### 5.3 时间记录（TimeRecord）
```typescript
interface TimeRecord {
  id: string;              // 唯一标识
  taskId: string;          // 关联任务ID
  taskType: 'core' | 'side'; // 任务类型
  date: string;            // 日期（YYYY-MM-DD）
  duration: number;        // 时长（分钟）
  recordedAt: number;      // 记录时间戳
  isManual: boolean;       // 是否手动修正
}
```

### 5.4 计时会话（TimerSession）- 用于恢复计时状态
```typescript
interface TimerSession {
  taskId: string;          // 任务ID
  taskType: 'core' | 'side'; // 任务类型
  startTime: number;       // 开始时间戳
  isRunning: boolean;      // 是否运行中
}
```

## 六、技术要点

### 6.1 计时准确性
- 使用 `Date.now()` 时间戳计算，而非 setInterval 累加
- 页面可见性变化时（visibilitychange 事件），重新计算当前时长
- 考虑使用 Web Worker 处理后台计时逻辑

### 6.2 数据持久化
- 使用 LocalStorage 存储所有数据
- 定期自动保存（如每30秒）
- 提供数据导出/导入功能（JSON格式），便于备份和迁移

### 6.3 跨端扩展预留
- 采用响应式设计，适配移动端浏览器
- 数据模型设计考虑未来云端同步
- 技术栈选择支持 PWA 或跨端框架（如 React + Capacitor）
